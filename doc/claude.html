🔴 WiFi Modem Crash — Root Cause Analysis Razer Phone 2 / SDM845 / kernel 6.16.0-rc2

最可能 #1 rmtfs 未提供 fsg 分區

Modem root_pd 啟動時向 rmtfs 請求 /boot/modem_fsg_oem_1 和 _2。若 rmtfs 拒絕，modem NV 初始化失敗後主動送出 fatal。

現況：rmtfs 只掛了 modemst1 / modemst2，沒有 fsg。160ms 的 crash 時間點正好是 modem EFS 初始化窗口。

sudo rmtfs \
  -P /dev/disk/by-partlabel/modemst1 \
  -P /dev/disk/by-partlabel/modemst2 \
  -P /dev/disk/by-partlabel/fsg &   # ← 加這行
先確認分區存在：ls /dev/disk/by-partlabel/ | grep -i fsg
最可能 #2 DTS rmtfs_mem 缺少 guard pages

SDM845 XPU 已知 bug：若記憶體分配到 rmtfs_mem 相鄰位置，XPU 會觸發 violation → crash，表現為 modem fatal。所有已 upstream 的 SDM845 裝置都有加 guard。

現況：aura DTS 裡需確認 rmtfs_mem 地址（從 android-fdt/ 取得），並套用新版寫法：

rmtfs_mem: rmtfs-region@ADDR {
    compatible = "qcom,rmtfs-mem";
    reg = <0 0xADDR 0 0x202000>;
    qcom,use-guard-pages;         /* ← 一行搞定 guard */
    no-map;
    qcom,client-id = <1>;
    qcom,vmid = <QCOM_SCM_VMID_MSS_MSA>;
};
中 #3 rmtfs 1.1.1 在 kernel 6.16 有已知 bug

Debian bug #1117938：rmtfs 1.1.1 在 SDM845 + kernel 6.16-sdm845 上會讓 WiFi/modem 失效，報告者使用的 kernel 版本與本專案完全相同。

確認版本：dpkg -l rmtfs。若是 1.1.1，考慮 downgrade 至 1.1。

中 #4 SLPI 未啟動（modem 可能依賴）

部分 SDM845 modem firmware 在 root_pd init 時需要 SLPI (remoteproc3) 提供 sensor bus services，否則在初始化 160ms 內 crash。

echo start | sudo tee /sys/class/remoteproc/remoteproc3/state
sleep 1
echo start | sudo tee /sys/class/remoteproc/remoteproc0/state
🔎 建議測試順序
#動作預期效果複雜度
1 rmtfs 加上 -P fsg 消除 modem NV init 時 rmtfs request 被拒 → fatal ⚡ 一行指令
2 DTS rmtfs_mem 確認地址 + 加 guard pages 消除 XPU violation 觸發的隱性 crash 需重編 kernel
3 確認 rmtfs 版本，若 1.1.1 則 downgrade 排除版本 regression ⚡ apt
4 先啟動 SLPI (remoteproc3) 再啟動 modem 滿足 modem 對 sensor bus 的依賴 ⚡ 一行指令
🛠 額外 debug 指令

啟動前開啟 MSS loader dynamic debug，可能暴露更多 crash 前的 kernel 訊息：

echo "file qcom_q6v5_mss.c +p" | sudo tee /sys/kernel/debug/dynamic_debug/control
echo "file qcom_q6v5.c +p"     | sudo tee /sys/kernel/debug/dynamic_debug/control

然後觀察 dmesg -w 輸出，留意 crash 前最後幾行的 function name。
