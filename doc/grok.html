🔴 Razer Phone 2 (SDM845) — Modem Crash 分析與解法

最後更新：2026-05-12　|　問題：root_pd 160~170ms fatal error without message（WLFW service 未出現）

核心問題

Modem root_pd 啟動後快速 fatal crash，service 66/43 註冊成功，但 service 69 (WLFW) 從未出現 → wlan_pd ELF 未載入。
ADSP/CDSP 正常，tqftpserv 運行但無 TFTP 請求。

🚨 可能原因（優先序）

優先原因說明
最高EFS / rmtfs 問題缺少 fsg overlay 或 EFS 讀取失敗，modem init 需要 NV 資料
高韌體版本不匹配wlanmdsp.mbn、modem.mbn、board-2.bin 與 Razer variant 不符
中DTS reserved-memory / SMMU / MSA 配置錯誤與 Android DTB 差異（rmtfs-mem、pil regions）
中Clock / Regulator 時序問題主線 MSS loader 未完全符合 Android 行為
低pd-mapper / QRTR 時序或 SLPI 未啟動部分 firmware 依賴 sensor PD

✅ 建議執行順序

優先動作目的
最高 強化 rmtfs + fsg overlay 從 Android 提取完整 EFS（modemst1/2 + fsg）並 overlay
高 啟用詳細 logging IPC logging + dynamic debug（見下方指令）
中 啟動 SLPI (remoteproc3) 確認 modem 是否需要 sensor service
中 比對 Android DTB 檢查 reserved-memory、clocks、pil-info 差異
低 更新 firmware + board-2.bin 使用 linux-firmware 最新 WCN3990 檔案

重要 Logging 指令

# Dynamic debug
echo 'module qcom_q6v5_mss +p' > /sys/kernel/debug/dynamic_debug/control
echo 'module remoteproc +p' > /sys/kernel/debug/dynamic_debug/control

# IPC logging
ls /sys/kernel/debug/ipc_logging/
cat /dev/ipc_logging/*   # 或對應 node

# 狀態檢查
qrtr-lookup
cat /sys/class/remoteproc/remoteproc0/state

💡 其他解法與參考

  • 確認 kernel 已開啟 CONFIG_QCOM_RMTFS_MEM、CONFIG_QCOM_Q6V5_MSS 等
  • 參考 msm8998/sdm845 mainline（OnePlus 6、Poco F1）與 postmarketOS 經驗
  • 試加入 DT 屬性 qcom,no-msa 或調整 MSA memory region
  • 若可能，從 Android 取得有 debug policy 的 modem coredump 看 crash reason
