⚡ 刷機流程

# 1. 進入 Fastboot
adb reboot bootloader
# 或 關機後按住 音量下 + 電源

# 2. 停用 Verified Boot
fastboot --disable-verity --disable-verification flash vbmeta vbmeta_disabled.img

# 3. 刷入 Boot Image (kernel + initramfs)
fastboot flash boot boot.img

# 4. 刷入根文件系統 (⚠️ 會清除 userdata！)
fastboot flash userdata rootfs-sparse.img

# 5. 重啟
fastboot reboot

⚠️ 映像檔路徑: output/boot.img, output/rootfs-sparse.img, output/vbmeta_disabled.img

🔴 WiFi Debug — 嘗試過的失敗方向

失敗 #1 缺少 tqftpserv

懷疑 WiFi modem (wlan_pd) 載入失敗是因為 AP 沒有提供 TFTP 服務。
處理: 從 andersson/tqftpserv 建置 ARM64 binary，安裝到裝置，建立 /readonly/firmware/image/wlanmdsp.mbn。
結果: tqftpserv 啟動成功（QRTR service 4096 可見），但 從未收到任何 TFTP 請求，modem 仍在 160ms crash。

失敗 #2 pd-mapper .jsn symlink 路徑錯誤

pd-mapper 找不到 *.jsn 檔案（路徑不符合 qmi_instance_id）。
處理: 在 /lib/firmware/ 建立 7 個 symlink 指向正確路徑。
結果: pd-mapper 正常啟動，modem 啟動速度提升（160ms crash），但 crash 問題未解決。

失敗 #3 懷疑是 wlan_pd (modemuw.jsn) 問題

Binary search：移除 /lib/firmware/modemuw.jsn，觀察 modem 是否因 wlan_pd 設定問題 crash。
結果: 移除後 crash 時間、行為 完全相同（161ms）。
結論: crash 在 modem root_pd，與 wlan_pd 無關。

失敗 #4 停用 pd-mapper

懷疑 pd-mapper 回應觸發 crash。停用 pd-mapper 後再啟動 modem。
結果: crash 時間 完全相同，pd-mapper 不是 crash 原因。

🔍 目前已確認的事實

  • Modem 啟動後約 160~170ms 發生 fatal error without message（不是 watchdog，是 modem 主動送出 fatal signal）
  • Modem 在 crash 前成功註冊了 service 66 (servreg) 和 service 43 (subsystem control) on node 0
  • Service 69 (WLFW/ath10k) 從未出現，代表 wlan_pd ELF 根本還沒被載入
  • ADSP (node 5) 和 CDSP (node 10) 正常運行，問題只在 modem
  • 韌體路徑正確（mba.mbn, modem.mbn 都能讀取）
  • Coredump 設定: disabled（echo coredump 回報 Invalid argument，無法取得 crash dump）
  • MBA booted without debug policy = 正式版韌體，無 crash reason string
  • rmtfs 目前以 modemst1/modemst2 分區執行，但沒有 fsg overlay 檔案

🔎 待確認的方向

優先級方向理由
高 確認 rmtfs EFS mount 是否成功 Modem 在 root_pd init 時需要 EFS，rmtfs 沒有 fsg overlay 可能導致 NV 讀取失敗後 crash
高 查看 IPC logging (/sys/kernel/debug/ipc_logging/) Modem 可能透過 SMEM 留下內部 log，包含 crash 前的函式呼叫
中 啟用 QCOM dynamic debug (module qcom_q6v5_mss +p) 可能暴露更多 MSS loader 的細節訊息
中 啟動 SLPI (remoteproc3) 部分 modem firmware 依賴 SLPI sensor service
低 比對 Android kernel modem_q6v5 驅動差異 Android 可能對 MSA memory / clock / PIL 有額外設定

🚀 服務手動啟動順序

# 1. pd-mapper (通常已 systemd 自啟)
sudo systemctl start pd-mapper

# 2. rmtfs (EFS partition 服務)
sudo rmtfs -P /dev/disk/by-partlabel/modemst1 \
           -P /dev/disk/by-partlabel/modemst2 &

# 3. tqftpserv (wlan_pd 韌體 TFTP)
LD_LIBRARY_PATH=/usr/lib/aarch64-linux-gnu /tmp/tqftpserv -d \
    > /tmp/tqftpserv.log 2>&1 &

# 4. Load kernel modules
cd /tmp
sudo insmod qrtr.ko; sudo insmod qrtr_smd.ko
sudo insmod qcom_pil_info.ko; sudo insmod qcom_sysmon.ko
sudo insmod qcom_q6v5.ko; sudo insmod qcom_mdt_loader.ko
sudo insmod reset-qcom-pdc.ko; sudo insmod qcom_q6v5_pas.ko
sudo insmod qcom_q6v5_mss.ko

# 5. 啟動 modem
echo start | sudo tee /sys/class/remoteproc/remoteproc0/state

# 確認狀態
qrtr-lookup
cat /sys/class/remoteproc/remoteproc0/state
