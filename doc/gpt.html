🔍 Razer Phone 2 — WiFi / Modem Crash 分析

目前資訊顯示： 問題不在 ath10k / wlan firmware / tqftpserv， 而是在 modem root_pd early init 階段 crash。

📌 已確認的重要事實

  • Crash 發生於 modem 啟動後約 160~170ms
  • fatal error without message = firmware 主動 fatal
  • service 43、service 66 已成功註冊
  • service 69 (WLFW) 從未出現
  • tqftpserv 從未收到 request
  • wlan_pd 尚未開始 load
  • ADSP / CDSP 正常
  • 問題集中於 modem root_pd init

🧠 最可能原因（依機率排序）

優先 可能原因 理由
高 rmtfs / EFS / NV storage 不完整 Qualcomm modem 啟動時需 mount EFS、載入 NV/calibration。
缺少 fsg/fsc、EFS 損壞、NV checksum fail 都可能在 root_pd early init 直接 fatal。
高 Android downstream 額外 PAS/PIL 初始化 主線 kernel 可能缺：
  • SCM auth sequence
  • MSA memory setup
  • XPU/protection config
  • clock/regulator votes
  • DMA/cache coherency handling
症狀非常像 shared memory / DMA 問題。
中 缺少 SLPI dependency 部分 Qualcomm modem firmware 依賴 SLPI/SSC service。
若 SLPI 未啟動，modem 可能直接 abort。
中 board-2.bin / RF calibration mismatch 若 calibration blob 或 board data 缺失， modem early init 可能直接失敗。
低 6.16-rc2 upstream regression 近期 qcom remoteproc / WCN3990 / ath10k upstream 仍持續修正中。

🔥 最可疑點：EFS / rmtfs

目前最像的情況： 

root_pd start
 → attach remote FS
 → mount EFS
 → load NV items
 → init RF
 → spawn wlan_pd

你目前 crash 發生於： wlan_pd 尚未 spawn 前。

而你目前：

  • 只有 modemst1 / modemst2
  • 沒有 fsg overlay
  • 沒有 fsc

這在 Qualcomm device 上非常可疑。

🧪 建議立即測試

1️⃣ 啟動 SLPI

echo start \
 | sudo tee /sys/class/remoteproc/remoteproc3/state

再重新啟動 modem。

2️⃣ 確認 Android partitions

adb shell ls -l /dev/block/by-name/

確認是否存在：

  • fsg
  • fsc

3️⃣ rmtfs 加入 fsg/fsc

rmtfs \
  -r /dev/disk/by-partlabel/fsg \
  -s /dev/disk/by-partlabel/fsc \
  -P /dev/disk/by-partlabel/modemst1 \
  -P /dev/disk/by-partlabel/modemst2

4️⃣ 開 dynamic debug

echo 'module qcom_q6v5_mss +p' \
 | sudo tee /sys/kernel/debug/dynamic_debug/control

5️⃣ IPC logging

ls -R /sys/kernel/debug/ipc_logging/

可能留下：

  • SSR reason
  • PIL stage
  • subsystem failure point

📌 我目前的判斷

目前不像：

  • ath10k bug
  • wlan firmware bug
  • tqftpserv bug

而更像：

  • EFS / NV mount failure
  • DMA/shared memory 問題
  • 缺 subsystem dependency
  • downstream PAS 初始化缺失

其中： EFS/rmtfs 問題 + downstream memory handling 機率最高。
